Discuss the scope of var, let, and const

var, let and const are JavaScript keywords which are used for declaring variables. These have different types of scope.

var is function scoped. That means if we declare a variable inside a function, the variable can only be accessible in that function. It is not accessible outside the function. Trying to access it from outside the function will cause an error. But if we declare a variable inside a block using var, it will be accessible outside the block also.

let is block scoped. That means if we declare a variable inside a block using let, it can only be accessible inside that block. Trying to access from outside the block will cause an error.

const is also block scoped. It behaves the same as let.

Tell us the use cases of null and undefined

null means empty value. Sometimes some value may cause an unexpected behavior, then we reassign null for that. The type of null is 'object'.

undefined means the value is not available. If we initialize a variable but didn't assign any value, then the value of the variable is undefined. If we didn't return any value from a function, then the value of that function is undefined. When any property does not exist in an object, the property value is undefined. The type of undefined is 'undefined'.

What do you mean by REST API?

REST means Representational State Transfer. API means Application Programming Interface. It is a stateless API. It is simple, scalable and compatible with http.

It supports CRUD operation i.e. create, read, update and delete. Some methods are used for these operations such as get, post, put, delete etc.
